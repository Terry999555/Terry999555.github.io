Java 回掉

Java回掉机制

模块之间总是存在着一定的接口，从调用方式上看，分为三类：

同步调用、异步调用、回调。

  1. 同步调用
    同步调用是最基本并且最简单的一种调用方式，类A的方法a()调用类B的方法b()，一直等待b()方法执行完毕，a()方法继续往下走。这种调用方式适用于方法b()执行时间不长的情况，因为b()方法执行时间一长或者直接阻塞的话，a()方法的余下代码是无法执行下去的，这样会造成整个流程的阻塞。

  1. 异步调用
    异步调用是为了解决同步调用可能出现阻塞，导致整个流程卡住而产生的一种调用方式。类A的方法方法a()通过新起线程的方式调用类B的方法b()，代码接着直接往下执行，这样无论方法b()执行时间多久，都不会阻塞住方法a()的执行。但是这种方式，由于方法a()不等待方法b()的执行完成，在方法a()需要方法b()执行结果的情况下（视具体业务而定，有些业务比如启异步线程发个微信通知、刷新一个缓存这种就没必要），必须通过一定的方式对方法b()的执行结果进行监听。

  1. 回掉
  • 类A的a()方法调用类B的b()方法
  • 类B的b()方法执行完毕主动调用类A的callback()方法

  • 双向的调用方式

    老师向学生提问问题，学生思考后回答老师的问题。

  • 首先要有回答老师问题的方式、渠道。所以我们要定义一个接口，用来实现老师获得学生
    答案。

  • 老师要有学生的引用，就是老师让那一位同学来回答问题，学生可以抽离出来，作为一个接口，用来回答老师的问题，任何人都可以实现这个接口，来回答问题。

  • 老师要获得答案就要实现接口，并且想要学生回答问题，就要有学生的引用和问题。然后提问的方法中调用学生回答问题的方法。

  • 具体学生(Tom)的思考和回答问题的方法。


我们可以实现，老师想学生提出问题，学生思考几秒钟后，回答老师的问题，并回掉接口，把答案传递给老师。

回掉的核心–>就是回调方将本身即this传递给调用方

疑问

  1. 使用同步调用的方式，学生对象回答完毕问题之后直接把回答的答案返回给老师对象

    这个是没有问题的。

  • （1）如果老师想要的数据越来越多，那么返回的对象得越来越大，而使用回调则可以进行数据分离，将一批数据放在回调方法中进行处理，至于哪些数据依具体业务而定，如果需要增加返回参数，直接在回调方法中增加即可。
  • 无法解决老师希望得到学生姓名、学生解题思路先于学生回答的答案的问题。无法解决先后的顺序，回掉可以同步不同的方法和顺序实现，先后。

异步回掉
